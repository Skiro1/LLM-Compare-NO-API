📡 API Документация

Добро пожаловать в документацию API Multi-LLM. API предоставляет единый интерфейс для работы с различными языковыми моделями через единый формат запросов, совместимый с OpenAI API.

💡 Единый формат

Все провайдеры используют формат, совместимый с OpenAI API. Это позволяет легко переключаться между моделями без изменения кода.

Основные возможности

  • Доступ к 20+ языковым моделям через единый API
  • Поддержка чат-формата с системными промптами
  • Потоковые ответы (streaming)
  • JSON mode для структурированных ответов
  • Совместимость с OpenAI SDK

Ваш первый запрос

Вот минимальный пример отправки сообщения модели:

cURL
curl https://api.openai.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-4",
    "messages": [
      {"role": "system", "content": "Ты полезный ассистент."},
      {"role": "user", "content": "Привет! Расскажи о себе."}
    ]
  }'
import requests

response = requests.post(
    "https://api.openai.com/v1/chat/completions",
    headers={
        "Content-Type": "application/json",
        "Authorization": "Bearer YOUR_API_KEY"
    },
    json={
        "model": "gpt-4",
        "messages": [
            {"role": "system", "content": "Ты полезный ассистент."},
            {"role": "user", "content": "Привет! Расскажи о себе."}
        ]
    }
)

data = response.json()
print(data["choices"][0]["message"]["content"])
const response = await fetch("https://api.openai.com/v1/chat/completions", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": "Bearer YOUR_API_KEY"
  },
  body: JSON.stringify({
    model: "gpt-4",
    messages: [
      {role: "system", content: "Ты полезный ассистент."},
      {role: "user", content: "Привет! Расскажи о себе."}
    ]
  })
});

const data = await response.json();
console.log(data.choices[0].message.content);

🔑 Аутентификация

Все запросы к API требуют аутентификации через API ключ. Ключ передается в заголовке Authorization.

HEADER Authorization: Bearer YOUR_API_KEY

Замените YOUR_API_KEY на ваш реальный API ключ от провайдера.

Получение API ключей

Каждый провайдер имеет свой способ получения API ключа:

Провайдер Где получить ключ Формат ключа
OpenAI platform.openai.com sk-...
Anthropic console.anthropic.com sk-ant-...
Google aistudio.google.com AIza...
DeepSeek platform.deepseek.com sk-...
Mistral console.mistral.ai ...
Groq console.groq.com gsk_...
Hugging Face huggingface.co hf_...
⚠️ Безопасность

Никогда не храните API ключи в клиентском коде. Используйте переменные окружения или серверные прокси для production приложений.

💬 Chat Completions

Создает ответ модели для заданного разговора.

POST /v1/chat/completions

Параметры запроса

model string Обязательный
ID модели для использования. Например: gpt-4, claude-3-opus, gemini-pro
messages array Обязательный
Список сообщений, представляющих разговор. Каждый объект содержит role и content.
temperature number Опциональный
Температура выборки от 0 до 2. Более высокие значения делают ответы более случайными. По умолчанию: 1
max_tokens integer Опциональный
Максимальное количество токенов в ответе. По умолчанию: модель-зависимое
stream boolean Опциональный
Включить потоковую передачу ответа. По умолчанию: false
response_format object Опциональный
Формат ответа. Используйте {"type": "json_object"} для JSON mode

Структура сообщений

Каждое сообщение в массиве messages содержит:

Роль Описание
system Системный промпт, задающий поведение модели
user Сообщение от пользователя
assistant Предыдущий ответ модели (для контекста)

Пример ответа

{ "id": "chatcmpl-abc123", "object": "chat.completion", "created": 1699876543, "model": "gpt-4", "choices": [ { "index": 0, "message": { "role": "assistant", "content": "Привет! Я языковая модель, разработанная OpenAI..." }, "finish_reason": "stop" } ], "usage": { "prompt_tokens": 25, "completion_tokens": 150, "total_tokens": 175 } }

📋 List Models

Возвращает список всех доступных моделей.

GET /v1/models

Не требует параметров. Возвращает список всех моделей, доступных для вашего API ключа.

Пример запроса

curl https://api.openai.com/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"
import requests

response = requests.get(
    "https://api.openai.com/v1/models",
    headers={"Authorization": "Bearer YOUR_API_KEY"}
)

models = response.json()["data"]
for model in models:
    print(model["id"])

Пример ответа

{ "object": "list", "data": [ { "id": "gpt-4", "object": "model", "created": 1687882411, "owned_by": "openai" }, { "id": "gpt-3.5-turbo", "object": "model", "created": 1677610602, "owned_by": "openai" } ] }

🔍 Retrieve Model

Возвращает информацию о конкретной модели.

GET /v1/models/{model}

Параметры пути

model string Обязательный
ID модели. Например: gpt-4

Пример запроса

curl https://api.openai.com/v1/models/gpt-4 \
  -H "Authorization: Bearer YOUR_API_KEY"

⚡ Streaming

Потоковая передача позволяет получать ответ модели по мере его генерации, улучшая воспринимаемую скорость ответа.

💡 Когда использовать streaming

Используйте streaming для длинных ответов или когда важно показать пользователю прогресс генерации в реальном времени.

Включение streaming

Добавьте параметр "stream": true в запрос:

curl https://api.openai.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-4",
    "messages": [{"role": "user", "content": "Расскажи длинную историю"}],
    "stream": true
  }'
import requests

response = requests.post(
    "https://api.openai.com/v1/chat/completions",
    headers={
        "Content-Type": "application/json",
        "Authorization": "Bearer YOUR_API_KEY"
    },
    json={
        "model": "gpt-4",
        "messages": [{"role": "user", "content": "Расскажи длинную историю"}],
        "stream": True
    },
    stream=True
)

for line in response.iter_lines():
    if line:
        line = line.decode("utf-8")
        if line.startswith("data: ") and line != "data: [DONE]":
            import json
            data = json.loads(line[6:])
            content = data["choices"][0]["delta"].get("content", "")
            print(content, end="", flush=True)
const response = await fetch("https://api.openai.com/v1/chat/completions", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": "Bearer YOUR_API_KEY"
  },
  body: JSON.stringify({
    model: "gpt-4",
    messages: [{role: "user", content: "Расскажи длинную историю"}],
    stream: true
  })
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  const chunk = decoder.decode(value);
  const lines = chunk.split("\n").filter(l => l.startsWith("data: "));
  
  for (const line of lines) {
    if (line === "data: [DONE]") break;
    const data = JSON.parse(line.slice(6));
    const content = data.choices[0].delta.content || "";
    process.stdout.write(content);
  }
}

Формат SSE ответа

Каждый chunk приходит в формате Server-Sent Events:

data: {"id":"chatcmpl-123","object":"chat.completion.chunk","choices":[{"delta":{"content":"Привет"},"index":0}]}

data: {"id":"chatcmpl-123","object":"chat.completion.chunk","choices":[{"delta":{"content":" мир!"},"index":0}]}

data: [DONE]

❌ Ошибки

API использует стандартные HTTP коды ошибок. При возникновении ошибки тело ответа содержит детальное описание.

Код Описание Решение
400 Bad Request Проверьте формат запроса и параметры
401 Unauthorized Проверьте API ключ
403 Forbidden Нет доступа к запрашиваемому ресурсу
404 Not Found Проверьте URL и ID модели
429 Rate Limited Превышен лимит запросов. Подождите и повторите
500 Server Error Ошибка на стороне сервера. Повторите позже
503 Service Unavailable Сервис временно недоступен

Формат ошибки

{ "error": { "message": "Incorrect API key provided: sk-xxx...xxx", "type": "invalid_request_error", "param": null, "code": "invalid_api_key" } }

Retry логика

✅ Best Practice

При получении ошибки 429 (Rate Limit) или 5xx (Server Error), используйте exponential backoff: ждите 1 секунду, затем 2, 4, 8... Максимум 5 попыток.

⏱️ Rate Limits

Лимиты запросов зависят от провайдера и вашего тарифного плана. Информация о лимитах передается в заголовках ответа.

Заголовки ответа

Заголовок Описание
x-ratelimit-limit-requests Максимальное количество запросов в период
x-ratelimit-remaining-requests Оставшееся количество запросов
x-ratelimit-reset-requests Время сброса лимита (RFC 7231)
x-ratelimit-limit-tokens Максимальное количество токенов в период
x-ratelimit-remaining-tokens Оставшееся количество токенов

🤗 Hugging Face Inference API

Hugging Face предоставляет доступ к тысячам открытых моделей через единый облачный API. Формат запросов совместим с OpenAI API.

Пример запроса к HF

curl https://router.huggingface.co/hf-inference/models/meta-llama/Llama-3.3-70B-Instruct/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer hf_YOUR_TOKEN" \
  -d '{
    "model": "meta-llama/Llama-3.3-70B-Instruct",
    "messages": [{"role": "user", "content": "Привет!"}]
  }'

⚠️ Бесплатный тариф HF

Бесплатный Inference API имеет ограничения: крупные модели могут быть недоступны или работать медленно. Для стабильного использования рассмотрите PRO-тариф или Serverless Inference.
